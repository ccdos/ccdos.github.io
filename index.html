完成本文任務前，您必須要先裝備：

  • 版本控管技能
  • 知道什麼是 branch
  • Github 帳號一枚

了解 Github page

寫好了專案我們通常會需要一台 Server 上傳我們的網站。有很多付費的，也有免費的，但如果只是很單純的 html、js、css，那麼你可以使用 Github Pages 。通常，如果租用了主機，通常還會需要買一個域名（你不會想給用戶看到 ip 吧），很幸運地 Github Pages 還提供了一個免費的域名給我們。

建立一個 Repository

登入 Github，然後右上角的 + 然後點 New repository

接著這邊要注意，Repository name 這邊一定要填 [你的github帳號].github.io，像我的帳號是 sexyoung，所以我就要輸入 sexyoung.github.io，然後按 [Create Repository] 。

接著就可以看到建立完成的畫面。

然後它告訴我們可以透過 HTTPS or SSH 下載該 repo，接著我們打開 Terminal 並透過 HTTPS 的方式下載。

git clone https://github.com/你的帳號/你的帳號.github.io.git

編輯專案並推上去

進到該目錄並打開編輯器，建立一個 index.html 並打一些內容

然後把新增的 index.html commit 進去：

git add .

然後 push 它

git push

瀏覽器打 https://你的帳號.github.io 試試

剛剛編輯好的 index.html 已看到內容，接下來要新增其他檔案就以此類推。

新增其他專案目錄

通常我們把 你的帳號.github.io 當作主頁，未來的其他專案時，可能會放在 你的帳號.github.io/other-project

直接在 你的帳號.github.io 裡新增一個 other-project 目錄

接著把它推進 repo, 上傳。然後到 https://你的帳號.github.io/other-project 即可看見剛剛的東西。

這邊看起來應該是大工告成了，但…

隨著專案數量越來越多，你的帳號.github.io 目錄下也越來越多 其他專案資料夾 。而每次 commit 時，會發現參雜了其他 project-1 到 project-N 的 commit，而你的帳號.github.io 也不再 只有 放 你的帳號.github.io 的東西。理應來說， project-1 與 project-N 應該要有各自的 Repository，commit 資訊也應該個別存在獨立的專案，而不是全部集中於一個專案。

建立其他專案Repository

到 https://github.com/new 去建立一個名為 project-1 的 repo。

下載 project-1 專案並建立 index.html內容隨意。

接著 commit 它。

建立 gh-pages branch

git checkout -b gh-pages

下這個指令來建立 gh-pages branch 並 checkout 過去

直接 push gh-pages branch

打開瀏覽器，輸入 https://[你的帳號].github.io/project-1

這樣就可以把本來存在於原 主頁專案下的 其他專案 刪除了！

原專案有 project-1 資料夾，也有 project-1 的 gh-pages branch，會以哪個為主?

答案是以 project-1 gh-pages branch 為主，所以這樣就更沒有理由要放在原專案目錄下。

再讓我們看一下 主頁專案 與 project-1專案

保持只有該目錄專屬的檔案，這樣看起來清楚多了！

學習回顧

  • 利用 Github Pages 上傳個人主頁到 [你的帳號].github.io，即可在 https://[你的帳號].github.io 看到
  • 利用其他 repo 新增 gh-pages branch 將其顯示於https://[你的帳號].github.io/[其他專案名稱]
